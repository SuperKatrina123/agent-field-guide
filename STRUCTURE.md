## 1. 编排原则

这套知识库采用 `认知主题型` 作为主目录，而不是严格的生命周期编排。

原因有三点：

- 对已有 Agent 基础的开发者来说，最缺的通常不是“搭建步骤”，而是稳定的概念框架
- Agent 设计中的很多问题是横切问题，例如上下文、记忆、评测、安全，不适合只放在线性流程里
- 主题型结构更适合长期积累，后续补案例、模板、反模式和前沿内容时不容易失控

> [!tip] 核心原则
> 主目录负责稳定认知结构，学习路径负责降低阅读门槛，问题索引负责支持快速排障和回查。

- `01-重新建立Agent认知地图`

作用：
- 划清 Agent、Workflow、Prompt、上下文工程的边界

- `04-上下文结构设计`

## 3. 推荐阅读顺序

### 路径一：系统认知优先
- `01 -> 02 -> 03 -> 04 -> 05 -> 08 -> 10 -> 11`

适合人群：
- 已做过一些 Agent 实验
- 但对系统设计边界仍然模糊

### 路径二：搭建与架构优先
- `02 -> 03 -> 04 -> 06 -> 07 -> 08 -> 09`

适合人群：
- 已经明确要做 Agent
- 想快速判断工具、记忆、规划和协作怎么设计

### 路径三：工程与稳定性优先
- `10 -> 11 -> 12 -> 13`

适合人群：
- 系统已经在跑
- 当前更关心稳定性、评测、安全和成本

## 4. 页面角色分工

### 首页
负责总导航与路径入口。

### 主线页
负责建立概念框架、方法和判断口径。

### 索引页
负责按问题或场景组织跳转。

### 模板页
负责把方法落为可执行的设计文档、评测文档和检查清单。

### 附录页
负责维护术语、缩写和参考资料。

## 5. 当前落地策略

> [!info] 当前策略
> 第一阶段先搭建 `一级主题骨架 + 导航页 + 模板入口`，避免一开始就把正文写得过满。
> 等主干稳定后，再往每个主题下扩概念页、案例页和练习页。

### 第一阶段包含
- 主题型主目录
- 三张导航页
- 每个一级主题的骨架页
- 模板与附录入口

### 第二阶段建议补充
- 术语页网络
- 失败案例库
- 不同场景的设计模板
- 评测样例和观测样例
